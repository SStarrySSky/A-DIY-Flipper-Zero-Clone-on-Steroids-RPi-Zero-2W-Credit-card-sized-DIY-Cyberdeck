# PwnZero 走线优化方案

## 当前问题

### 1. 所有走线都是直线连到Pi Zero中心 (42.8, 27)
这会导致：
- 走线交叉严重
- 信号干扰
- 制造困难
- 不美观

### 2. 需要优化的走线组

**I2C总线 (SDA/SCL):**
- 从Pi Zero → OLED (31, 16)
- 从Pi Zero → PN532 (25, 5) [已修复位置]

**SPI总线 (MOSI/MISO/SCLK):**
- 从Pi Zero → CC1101 TX (11, 14)
- 从Pi Zero → CC1101 RX (11, 41)

**GPIO控制线:**
- 从Pi Zero → LED电阻 (75, 5/13/21/35)
- 从Pi Zero → 摇杆 (60, 27)

## 优化策略

### 1. 使用曼哈顿走线（Manhattan Routing）
- 只使用水平和垂直线段
- 避免45度角（除非必要）
- 减少过孔数量

### 2. 分组走线
- I2C走线走左侧
- SPI走线走左侧
- GPIO走线走右侧
- 电源走线走外围

### 3. 优化后的走线路径

**I2C到OLED:**
```
Pi Zero (42.8, 27)
→ 向左 (35, 27)
→ 向上 (35, 16)
→ 向左 (31, 16) OLED
```

**I2C到PN532:**
```
Pi Zero (42.8, 27)
→ 向左 (35, 27)
→ 向上 (35, 10)
→ 向左 (25, 10) PN532
```

**SPI到CC1101:**
```
Pi Zero (42.8, 27)
→ 向左 (20, 27)
→ 向上/下到目标
→ 向左 (11, Y) CC1101
```

**GPIO到LED:**
```
Pi Zero (42.8, 27)
→ 向右 (70, 27)
→ 向上/下到目标
→ 向右 (75, Y) 电阻
```

**GPIO到摇杆:**
```
Pi Zero (42.8, 27)
→ 向右 (55, 27)
→ 过孔到顶层
→ 向右 (60, 27) 摇杆
```
