# KiCad PCB 文件说明

## ❓ 为什么KiCad打开PCB是空的？

### 原因分析

KiCad的PCB文件（.kicad_pcb）目前只包含：
- ✅ 板框（85.6mm x 53.98mm）
- ✅ 4个安装孔
- ✅ 网络定义
- ❌ **没有实际的元器件footprint**

### 为什么没有元器件？

1. **KiCad工作流程**：
   - 正常流程：原理图 → 更新PCB → 元器件自动导入
   - 我们的情况：原理图是空的（只有文本注释）
   - 结果：PCB无法从原理图获取元器件

2. **手动创建PCB的局限**：
   - 在文本文件中手动编写完整的footprint定义非常复杂
   - 每个元器件需要几百行代码
   - 包括焊盘、丝印、封装尺寸等详细信息


## ✅ 推荐的解决方案

### 方案一：使用立创EDA（推荐）

**为什么推荐立创EDA？**
- 国内访问快
- 直接对接嘉立创PCB下单
- 元器件库完整（自动匹配LCSC料号）
- 操作简单

**步骤：**
1. 打开 https://lceda.cn/
2. 导入 `pwnzero.net` 网表文件
3. 导入 `pwnzero_bom.csv` BOM清单
4. 参考 `PCB_COMPLETE_DESIGN.md` 放置元器件
5. 布线
6. 直接下单


### 方案二：在KiCad中手动添加元器件

**步骤：**
1. 打开 `pwnzero.kicad_pcb`
2. 点击"添加封装"工具
3. 从库中选择每个元器件的封装
4. 参考 `PCB_COMPLETE_DESIGN.md` 放置到正确位置
5. 手动布线

**缺点：**
- 需要手动添加所有元器件（20+个）
- 需要手动连接所有网络
- 耗时较长


## 📋 项目文件说明

### 可用的设计文件

| 文件 | 用途 | 状态 |
|------|------|------|
| `pwnzero.net` | 网表文件 | ✅ 完整 |
| `pwnzero_bom.csv` | 物料清单 | ✅ 完整 |
| `PCB_COMPLETE_DESIGN.md` | 完整设计文档 | ✅ 完整 |
| `pwnzero.kicad_pcb` | KiCad PCB | ⚠️ 仅板框 |
| `pwnzero.kicad_sch` | KiCad原理图 | ⚠️ 仅文本 |


### 为什么提供这些文件？

1. **网表文件** - 通用格式，可导入任何EDA软件
2. **BOM清单** - 直接用于采购元件
3. **设计文档** - 包含所有坐标和连接信息
4. **KiCad文件** - 提供基础框架，可手动完善

## 🎯 总结

**最佳实践：使用立创EDA**
- 导入网表和BOM
- 自动匹配元器件
- 快速完成PCB设计
